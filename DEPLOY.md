# Как выложить сайт Vitolux в интернет (пошагово)

Сайт будет размещён на **Vercel** — бесплатно, без сервера, с автоматическим обновлением при каждом изменении кода.

---

## Часть 1. Подготовка: Git и GitHub

### Шаг 1.1. Установить Git (если ещё нет)

1. Откройте сайт: https://git-scm.com/downloads  
2. Скачайте установщик для вашей системы (Windows / macOS).  
3. Установите Git, оставив настройки по умолчанию.  
4. Проверьте: откройте **Терминал** (macOS/Linux) или **Command Prompt** (Windows) и введите:
   ```bash
   git --version
   ```
   Должна появиться версия, например: `git version 2.43.0`.

---

### Шаг 1.2. Зарегистрироваться на GitHub

1. Перейдите на https://github.com  
2. Нажмите **Sign up** и создайте аккаунт (email, пароль).  
3. Подтвердите email, если попросят.

---

### Шаг 1.3. Создать репозиторий на GitHub

1. Войдите в GitHub.  
2. В правом верхнем углу нажмите **+** → **New repository**.  
3. **Repository name:** например `vitolux-website` (латиница, без пробелов).  
4. Оставьте **Public**.  
5. **Не ставьте** галочки «Add a README» и «Add .gitignore» — репозиторий должен быть пустым.  
6. Нажмите **Create repository**.  
7. На следующей странице запомните адрес репозитория, например:  
   `https://github.com/ВАШ_ЛОГИН/vitolux-website.git`

---

### Шаг 1.4. Открыть папку проекта в терминале

1. Откройте **Терминал** (macOS) или **Command Prompt** / **PowerShell** (Windows).  
2. Перейдите в папку вашего проекта. Пример для macOS:
   ```bash
   cd "/Users/ievgenbrynza/Documents/Vitolux Украины"
   ```
   Для Windows (подставьте свой путь):
   ```bash
   cd C:\Users\ВашеИмя\Documents\Vitolux Украины
   ```

---

### Шаг 1.5. Инициализировать Git и отправить код на GitHub

Выполните команды **по очереди** (подставьте свой адрес репозитория и логин GitHub):

```bash
# 1. Инициализировать репозиторий в папке проекта
git init

# 2. Добавить все файлы
git add .

# 3. Сделать первый коммит
git commit -m "Initial commit: Vitolux website"

# 4. Переименовать ветку в main (если нужно)
git branch -M main

# 5. Указать ваш репозиторий на GitHub (ЗАМЕНИТЕ YOUR_USERNAME и vitolux-website на свои)
git remote add origin https://github.com/YOUR_USERNAME/vitolux-website.git

# 6. Отправить код на GitHub
git push -u origin main
```

При `git push` браузер или терминал попросят войти в GitHub (логин + пароль или токен).  
Если просят **пароль**, а у вас включена двухфакторная аутентификация, нужно использовать **Personal Access Token**:  
GitHub → Settings → Developer settings → Personal access tokens → Generate new token.  
Скопируйте токен и вставьте его вместо пароля при `git push`.

После успешного выполнения на странице репозитория на GitHub появятся все файлы проекта.

## Часть 2. Размещение на Vercel

### Шаг 2.1. Зарегистрироваться на Vercel

1. Откройте https://vercel.com  
2. Нажмите **Sign Up**.  
3. Выберите **Continue with GitHub** и разрешите Vercel доступ к вашему аккаунту GitHub.  
4. Заполните профиль при необходимости и нажмите **Continue**.

---

### Шаг 2.2. Импортировать проект

1. На главной странице Vercel нажмите **Add New…** → **Project**.  
2. В списке репозиториев найдите **vitolux-website** (или как вы назвали) и нажмите **Import**.  
3. Настройки оставьте как есть:
   - **Framework Preset:** Next.js (определится автоматически)
   - **Root Directory:** пусто
   - **Build Command:** `npm run build` (по умолчанию)
   - **Output Directory:** по умолчанию
4. Нажмите **Deploy**.

---

### Шаг 2.3. Дождаться сборки

1. Начнётся сборка (обычно 1–3 минуты).  
2. Появится лог: «Building…», затем «Deploying…».  
3. Когда увидите **Congratulations!** — сайт уже в интернете.

---

### Шаг 2.4. Открыть сайт в интернете

1. Нажмите **Visit** или скопируйте ссылку вида:  
   `https://vitolux-website-xxxx.vercel.app`  
2. Откройте её в браузере — это и есть ваш опубликованный сайт.

Эту ссылку можно отправлять клиентам. На бесплатном тарифе Vercel даёт:
- HTTPS (защищённое соединение)
- Глобальный CDN
- Автодеплой при каждом `git push` в ветку `main`

---

## Часть 3. Свой домен (необязательно)

Если хотите адрес вида `www.vitolux.com`:

1. Купите домен у регистратора (например reg.ru, nic.ua, Namecheap).  
2. В Vercel: зайдите в проект → **Settings** → **Domains**.  
3. Добавьте домен (например `www.vitolux.com`) и следуйте подсказкам Vercel: куда прописать DNS (A-запись или CNAME).  
4. После обновления DNS (от нескольких минут до 24 часов) сайт будет открываться по вашему домену.

## Краткая шпаргалка

| Действие              | Где / что делать                          |
|-----------------------|-------------------------------------------|
| Код в интернете       | GitHub (репозиторий)                      |
| Сайт доступен по URL  | Vercel (после Deploy)                      |
| Обновить сайт         | Изменить код → `git add .` → `git commit -m "описание"` → `git push` → Vercel сам пересоберёт и обновит сайт |

---

## Если что-то пошло не так

- **Ошибка при `git push`:** проверьте, что адрес `origin` правильный:  
  `git remote -v`  
  При необходимости:  
  `git remote set-url origin https://github.com/YOUR_USERNAME/vitolux-website.git`

- **Ошибка сборки на Vercel:** откройте в Vercel вкладку **Deployments** → последний деплой → **Building** и посмотрите лог ошибки. Чаще всего это опечатка в коде или отсутствующий файл — исправьте, сделайте коммит и пуш снова.

- **Сайт не открывается:** убедитесь, что деплой завершился зелёной галочкой (Success). Подождите 1–2 минуты и обновите страницу.

Если напишете, на каком шаге застряли (например: «не получается git push» или «Vercel пишет ошибку при сборке»), можно разобрать именно этот шаг подробнее.
